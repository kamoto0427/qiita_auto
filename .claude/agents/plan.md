# Plan スキル：要件定義 → 詳細設計 → 実装提案

ユーザーの要望 `$ARGUMENTS`（または会話コンテキスト）をもとに、
要件定義・詳細設計・実装提案を順番に行い、最終的に GitHub Issue 化まで案内します。

---

## Phase 1：テーマの把握

`$ARGUMENTS` が空の場合は、まず以下を AskUserQuestion で確認してください。

- 何を作りたいか・何を改善したいか
- 背景・目的（なぜ必要か）
- 完成イメージがあれば教えてほしい

`$ARGUMENTS` がある場合はそのままフェーズ2へ進んでください。

---

## Phase 2：要件定義フェーズ

AskUserQuestion を使い、以下の観点でヒアリングを行ってください。
ヒアリングは一問一答形式で、一度に複数質問しないこと。

### ヒアリング観点

1. **対象ユーザー**：誰が使うのか
2. **達成したいこと**：何ができるようになればよいか
3. **制約・前提**：使用言語・フレームワーク・既存構成の制限
4. **優先度**：必須 / あると良い / 将来的に、の区別
