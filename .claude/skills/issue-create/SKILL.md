## 手順
1. ユーザーの要望・機能アイデアをヒアリングし、実装内容を整理する
2. 現在のコードベースを調査し、変更が必要なファイルと実装方針を把握する
3. 以下の内容でIssueを作成する：
   - **タイトル**: `feat:` / `fix:` / `chore:` などのプレフィックスを付けた日本語タイトル
   - **本文**:
     - ## 概要（何をするか一言で）
     - ## 背景（なぜ必要か）
     - ## 機能詳細（具体的な仕様・箇条書き）
     - ## 変更ファイル（ファイル名と変更内容の表）
4. `gh issue create` コマンドでリポジトリ kamoto0427/qiita_auto にIssueを登録する
5. 作成されたIssueのURLをユーザーに報告する
